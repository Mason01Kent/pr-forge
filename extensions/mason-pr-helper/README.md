# mason-pr-helper

A private VS Code extension for generating PR summaries and reviews using DeepSeek AI.

## Commands

| Command | Description |
|---|---|
| `MasonDevTools: Initialize Project Config` | Scan workspace and create `.mason-devtools.json` |
| `MasonDevTools: Open Project Config` | Open `.mason-devtools.json` in the editor |
| `MasonDevTools: Generate PR Body` | Generate `.pr/PR_BODY.md` and `.pr/PR_TITLE.txt` |
| `MasonDevTools: Generate Full PR Review` | Generate `.pr/PR_BODY.md`, `.pr/PR_TITLE.txt`, and `.pr/PR_REVIEW.md` |

## Status Bar Buttons

- **$(tools) MasonDevTools** — Opens project config
- **$(git-pull-request) PR Body** — Generates PR body
- **$(comment-discussion) PR Review** — Generates full PR review

## Required Environment Variable

| Variable | Description |
|---|---|
| `DEEPSEEK_API_KEY` | Your DeepSeek API key (never stored in files) |

## Generated Files

All output is placed in the project's configured output directory (default `.pr/`):

| File | Generated By |
|---|---|
| `.pr/PR_TITLE.txt` | Generate PR Body, Generate Full PR Review |
| `.pr/PR_BODY.md` | Generate PR Body, Generate Full PR Review |
| `.pr/PR_REVIEW.md` | Generate Full PR Review only |

## Current Limitations

- No GitHub PR creation or commenting (planned for v2)
- No webview UI — uses commands, notifications, and output channel
- PowerShell script execution requires PowerShell 5.1+
- Single workspace folder only (no multi-root workspace support yet)

## How It Works

1. The extension detects the active workspace folder.
2. On command, it spawns `scripts/pr-helper/New-PrRequest.ps1` as a child process.
3. The PowerShell script gathers git info, runs tests, calls DeepSeek API, and writes output files.
4. stdout/stderr are streamed to the `MasonDevTools` output channel.
5. Generated files are opened automatically when complete.
